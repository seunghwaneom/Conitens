Conitens 에이전트 시스템 비교 리포트

Agentland, Maestro, Optio, Agent Squad, AutoGen, Claw3D, Pixel Agents, CLI-JAW를 현재 Conitens와 비교해 증거, 안전, 운영자 워크벤치 중심의 실행 백로그로 정리했다.

기준 커밋2c25a9e2 기반의 로컬 워크트리 비교

리포트 성격계획 산출물이며 이 조사 패스 자체는 제품 코드를 변경하지 않았다.

운영 정체성Conitens는 외부 AI 런타임을 대체하지 않는 event-first 운영/control plane이다.

기준선 핵심 판단 격차 매트릭스 우선순위 백로그 가드레일 소스 스냅샷

기준선

현재 런타임 진실은 계속 scripts/ensemble.py, .notes/, .agent/에 있다. .conitens/는 forward 계약 표면이며, 나중 배치에서 명시하기 전까지 기존 런타임을 대체한다고 주장하지 않는다.

핵심 판단

외부 시스템의 강점은 그대로 복제하지 않고 Conitens의 approval, verify, replay 경계에 맞춰 선택적으로 흡수한다.

Adopt

Agentland식 관측성

provider, model, token, cost, latency, tool, PII 필드를 event/projection 레이어부터 도입한다.

Adopt

Optio식 reconciler

작업 상태 전환을 pure decision으로 분리하고, 나중에 CAS와 periodic resync로 확장한다.

Adapt

Maestro와 CLI-JAW 운영 UX

세션 roster, doctor evidence, worktree-aware 메타데이터를 우선하고 remote control은 뒤로 둔다.

Adapt

Agent Squad와 AutoGen 패턴

새 런타임 dependency가 아니라 workflow/router/prompt 계약으로 번역한다.

Defer

Spatial UI 심화

Claw3D와 Pixel Agents는 presence와 layout 참고용이며 운영 워크벤치가 제품 중심이다.

Avoid

승인 우회

원격, 웹, MCP, 공간 UI 쓰기 경로는 기존 approval/verify gate를 약화하면 안 된다.

격차 매트릭스

축 참고 시스템 Conitens 상태 결정 권장 변경
Provider traffic observability Agentland append-only event와 replay는 있으나 provider-call telemetry schema가 부족하다. Adopt provider-call event contract를 먼저 추가하고 reverse proxy는 제외한다.
Cost and budget rollups Agentland, Maestro token compression과 task evidence는 있으나 runtime/model/task별 spend attribution이 약하다. Adopt run, task, agent, provider, model별 read-only cost summary projection을 추가한다.
Task/PR feedback loop Optio operator task, linked evidence, approval은 있으나 PR/CI watcher loop가 없다. Adapt auto-merge 전에 GitHub/CI read evidence와 human-approved resume suggestion을 붙인다.
Reconciliation loop Optio validator/retry/approval discipline은 있으나 stuck task repair decision이 bridge 내부에 흩어져 있다. Adopt operator_task_reconciler를 pure decision module로 분리한다.
Runtime/session operations Maestro, CLI-JAW background CLI session과 dashboard는 있으나 multi-runtime roster와 install evidence가 얇다. Adapt runtime/session roster와 doctor evidence artifact를 operator-facing으로 노출한다.
Intent routing Agent Squad, AutoGen workflow, room, replay, handoff는 있으나 classifier/router contract가 없다. Adapt routing decision을 evidence로 기록하는 workflow contract를 추가한다.

우선순위 백로그

P0 Evidence and Safety
  1. Provider-call telemetry event contract: provider, model, token, cost, latency, tool, PII 필드 추가.
  2. Operator task reconciler: task, approval, run, validation, handoff evidence 기반 pure decision 추가.
  3. Install/doctor evidence mode: 환경, CLI, PATH, version, auth posture, verification log를 redacted artifact로 출력.
P1 Operator Workbench
  1. Runtime/session roster: Codex, Claude, Gemini, OpenCode와 toolchain 상태를 read-only로 표시.
  2. PR/CI evidence ingestion: task detail에 GitHub/CI 읽기 evidence를 붙이고 resume suggestion만 제공.
  3. Persistent-agent wake model: wake source, per-turn record, inbox state를 기존 background/session adapter 뒤에 둔다.
  4. Router workflow contract: task intent를 role, room, workflow 후보로 분류하고 이유를 evidence로 기록.
P2 Spatial and Interaction
  1. Status-confidence and transcript watcher: heuristic status에 evidence ref와 confidence를 붙인다.
  2. Office layout import/export: asset manifest와 layout editor 아이디어를 보조 기능으로 제한한다.
  3. Keyboard-first command surface: run, task, agent, approval, evidence search 전환을 빠르게 만든다.

가드레일

금지

새 런타임 소유자 도입 금지

  • AutoGen core dependency를 추가하지 않는다.
  • Agent Squad runtime을 embedding하지 않는다.
  • Agentland full reverse proxy를 첫 slice에서 만들지 않는다.
금지

승인 약화 금지

  • approval-bypass launch control을 추가하지 않는다.
  • remote, web, MCP, spatial UI write path는 existing approval/verify gate를 따라야 한다.
  • auto-merge와 unattended resume은 별도 승인 정책 전까지 제외한다.
주의

민감 telemetry 처리

  • cost와 PII telemetry는 projection control과 redaction을 먼저 둔다.
  • raw prompt, completion, response content는 provider event payload와 dashboard summary에 노출하지 않는다.
  • active runtime truth는 scripts/ensemble.py, .notes/, .agent/로 유지한다.
